## 1. AWS 계정 결제정지 → 전 도메인 다운 복구

- **증상**: www뿐 아니라 api·app·admin 등 lighthouse.career 전 도메인 DNS 해석 실패.
- **원인**: AWS 계정 결제 미납으로 일시 정지. 장기 정지로 Route53이 DNS 서빙 중단(옛 NS `REFUSED`).
  IAM 키도 `InvalidClientTokenId`. 도메인 등록 자체는 정상(.career 위임 살아있음).
- **복구**: 미납금 결제 → 계정 복원. 복원이 IAM→Route53 순으로 전파되며 `list-hosted-zones`가 일시
  0개로 보였으나, 원본 hosted zone(`Z01901561U6SFBZRKNXN5`)이 옛 NS 그대로 자가 복구됨.
  (복원 도중 새 존을 만들었다가 중복이라 삭제함 — 교훈: 복원 직후 빈 것처럼 보여도 새 존 만들지 말 것.)
- **결과**: www 200 / apex 301→www / app 200 / test 200 / admin 307 / api /health 200. 전 도메인 정상.
- **주의**: 전 인프라가 단일 AWS 계정(706302944613)에 집중 → 결제 하나로 전 서비스 동시 다운. Billing 알림 점검 권장.
  도메인 이메일은 improvmx(MX/SPF).
