## [인프라] API 클라우드 이전 (홈서버 PM2 → AWS Lightsail 컨테이너)

홈서버 PC가 내려가 `api.lighthouse.career`가 CloudFront 504(오리진 무응답) 발생. 로그인·회원가입 전면 불가. 단일 장애점(홈서버) 제거를 위해 API를 클라우드로 이전.

**핵심 진단**: DB는 이미 MongoDB Atlas(클라우드), 파일은 S3(클라우드). 홈서버에 있던 건 **Express 앱 하나뿐** → 앱만 이전하면 됨(데이터 마이그레이션 불필요).

**선택**: AWS Lightsail 컨테이너(서울). App Runner는 서울 리전 미지원, 기존 배포 자격증명은 ECR/IAM 권한이 없어 Lightsail + ghcr.io 조합으로 결정.

작업 내용:
- **Dockerfile 슬림화**: ubuntu+pm2 cluster → 멀티스테이지 `node:20-slim`, 단일 `node server.js`, 비루트 실행, 포트 8080
- **.dockerignore 정비**: `dump/`·`*.tar`(258MB)·`docs/`·`scripts/` 등 런타임 비의존 대용량 제외
- **이미지 레지스트리**: `ghcr.io/gwon6482/lighthouse-api`(public). 로컬 Docker 부재로 CI가 빌드
- **Lightsail 컨테이너 서비스** `lighthouse-api`(ap-northeast-2, nano, scale 1) 생성·배포. env: NODE_ENV/PORT=8080/MONGODB_URI/JWT_SECRET(신규)/JWT_EXPIRES_IN. healthCheck `/api/health`
- **도메인 전환**: Lightsail 관리형 TLS 인증서 발급 → `api.lighthouse.career`를 (구)CloudFront(홈서버 오리진)에서 Lightsail로 CNAME 전환(Route53)
- **CI/CD 교체**: 기존 SSH→홈서버 pm2 워크플로 폐기 → `main push → Docker 빌드 → ghcr push → Lightsail 자동 재배포`(GitHub Secrets로 env 주입)

검증(도메인 기준): `/api/health` 200(Atlas connected), `check-email` 200, 실계정 `test5@test.com` 로그인 200(토큰+유저데이터 정상).

> ⚠️ JWT_SECRET을 신규 발급해 적용 → 기존 발급 토큰 무효화. 기존 로그인 사용자는 **1회 재로그인** 필요.
> ⏳ 후속: S3 사진 업로드 env 미설정(진로달성 인증사진 기능 사용 시 추가), 구 API용 CloudFront 배포 정리(권한 보류).

## [FE] SCSS `@import` deprecation 제거 (`@use`/`@forward` 전환)

lighthouse-test 렌더링 시 Dart Sass `@import` deprecation 경고 다수 발생(3.0에서 제거 예정).

- `appearance/styles.scss`: 변수/믹스인 + 전체 파셜 `@import` → `@use` 전환
- `_mixins.scss` 및 변수/믹스인 사용 파셜 13개 상단에 `@use 'variables' as *; @use 'mixins' as *;` 추가(@use는 전역이 아니므로 각 파일이 직접 임포트)
- 검증: `sass --load-path=appearance styles.scss` → EXIT 0, deprecation 경고 0
