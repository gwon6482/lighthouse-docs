## 배경

진로달성(career-achievement) 모듈은 진로계획·주간일정까지는 DB에 저장되지만, 사용자의 실제 달성 행위(완료 토글 / 인증사진·난이도·메모 / 커리큘럼 체크)는 전부 브라우저 localStorage 에만 남아 있었다. 기기·브라우저만 바뀌어도 모든 달성 기록과 인증사진이 소실되는 상태. 이를 서버 영속화하고, 인증사진은 base64 를 DB 에 넣지 않고 S3 업로드 후 URL 만 저장하도록 구현.

> 상태: **코드 구현 + 로컬 검증 완료 / 배포 대기** (DB API push, AWS 콘솔 셋업, 운영 env 설정 미완)

## [BE] 진로달성 기록 API + S3 presigned 업로드

**파일**: `models/AchievementRecord.js`, `models/CurriculumCompletion.js`, `config/s3.js`, `controllers/achievementController.js`, `routes/careerPlan.js`, `env.example`

### 신규 모델 (user_data DB)
- `achievement_records` — 일별 완료 토글 + 인증 기록(photoUrl/difficulty/note/elapsedSec/doneAt)을 1도큐먼트로 통합
  - 키: `(userUid, planId, date, itemType, itemId)` unique, 피드용 `(userUid, planId, doneAt desc)` 인덱스
- `curriculum_completions` — 커리큘럼 항목 체크. 키: `(userUid, planId, projectId, week, idx)` unique

### 신규 엔드포인트 (career-plan 하위, authenticate + 소유권 가드)
| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/api/career-plan/:planId/achievements?from=&to=` | 달성 기록 목록(완료상태 복원 + 피드) |
| PUT | `/api/career-plan/:planId/achievements/:date/:itemType/:itemId` | upsert (토글 또는 인증 기록) |
| DELETE | `/api/career-plan/:planId/achievements/:date/:itemType/:itemId` | 삭제 (토글 off) |
| GET | `/api/career-plan/:planId/curriculum` | 커리큘럼 완료 목록 |
| PUT | `/api/career-plan/:planId/curriculum/:projectId/:week/:idx` | 토글 (body.done=false 면 해제) |
| POST | `/api/career-plan/uploads/presign` | 인증사진 S3 presigned PUT URL 발급 |

### S3 업로드
- 의존성 추가: `@aws-sdk/client-s3`, `@aws-sdk/s3-request-presigner`
- `config/s3.js`: region `ap-northeast-2`, 자격증명은 env 또는 SDK 기본 체인(~/.aws / 인스턴스 프로파일)
- key 규칙: `uploads/achievements/{uid}/{uuid}.jpg`, presign 만료 5분
- 소유권 검증은 `weeklyScheduleController` 의 `ensureOwnedPlan` 패턴 재사용

## 남은 작업 (배포 전 선행)

| 항목 | 내용 |
|------|------|
| AWS 버킷 CORS | `lighthouse-career-fe` `uploads/` prefix 에 브라우저 PUT(앱 오리진) 허용 — presigned 직접 업로드용 (신규 필수) |
| 운영 env | `S3_UPLOAD_PUBLIC_BASE` = CloudFront 도메인 (버킷 퍼블릭 차단 → 직링크 403 회피) |
| IAM 확인 | `lighthouse-deploy` 가 해당 prefix `s3:PutObject` 보유 여부 (FE 배포 권한 이미 있어 충족 가능성) |
| CloudFront | `uploads/*` 전용 비헤이비어 (선택 — SPA 404 리라이트 회피·캐시) |
| 배포 | DB API main push → GitHub Actions 자동 배포 |

> 참고: 로컬 `aws` CLI 가 python@3.14/libexpat ABI 불일치로 깨져 있음(런타임 무관). 콘솔 확인 전 `brew reinstall awscli` 필요.
