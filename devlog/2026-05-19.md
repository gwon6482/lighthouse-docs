# 개발일지 2026-05-19

## DB API — 유저 관련 API 5단계 구현 완료

**Stage 1 — 인증 (Auth)**
- `POST /api/auth/register`, `POST /api/auth/login`, `POST /api/auth/logout`, `GET /api/auth/me`
- JWT Bearer Token (7일), bcrypt 해시, `middleware/auth.js` 구현

**Stage 2 — 프로필 (User Profile)**
- `GET/PUT /api/user/profile`, `DELETE /api/user`
- settings 중첩 병합, 소프트 삭제 (`isActive: false`)

**Stage 3 — 설문 결과 연결**
- `POST/GET /api/user/survey-results`

**Stage 4 — 북마크**
- `GET/POST/DELETE /api/user/bookmarks/:jobCode`
- job_data cross-join, Mongoose 모델 중복 컴파일 방지 패턴

**Stage 5 — 기기 토큰 (FCM)**
- `POST/DELETE /api/user/devices/:deviceId`, deviceId 기준 upsert

**기타**: Swagger bearerAuth 전역 적용, cross-env 적용(Windows npm dev), guide/ MD 문서 추가

## 인프라 — GitHub Actions 자동 배포 수정

**문제**: SSH 인증 반복 실패

**원인 및 해결**:
1. `PasswordAuthentication` 기본값 `no` → `/etc/ssh/sshd_config`에 `yes` 추가
2. `SSH_USER` Secret이 잘못된 계정 → `root`로 재설정
3. deploy.yml: `key` → `password` 인증으로 변경

**결과**: DB API + Admin 양쪽 `main` push → 자동 배포 정상 동작
