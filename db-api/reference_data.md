# 참조 데이터 (reference_data DB)

> DB: `reference_data` (2개 컬렉션, 총 441건)

두 컬렉션의 역할 구분:

| 컬렉션 | 역할 | 건수 | 용도 |
|--------|------|------|------|
| `survey_elements` | **설문 검사 요소** | 239건 | 설문 결과 해석 시 코드 → 이름/정의 매핑 |
| `career_attributes` | **직업 속성 코드** | 202건 | 진로백과(job_info)의 details 필드 코드 → 이름/정의 매핑 |

## 1. survey_elements — 설문 검사 요소 (239건)

설문 5개 파트(T1, T21, T22, T23, T3)의 요소 코드 정의 테이블.  
결과 보고서에서 "코드 → 한글 이름/정의" 변환에 사용.

### 도큐먼트 구조
```json
{
  "test_code": "T1",
  "area": "personality",
  "level": "upper",
  "code": "E",
  "name": "외향성",
  "definition": "외부 세계와의 상호작용에서 활력과 긍정적인 감정을 얻는 경향",
  "parent_code": null
}
```

| 필드 | 타입 | 설명 |
|------|------|------|
| `test_code` | String | T1 / T21 / T22 / T23 / T3 |
| `area` | String | personality / talent / interest / values / environmental |
| `level` | String | `upper` (상위) / `sub` (하위) / `item` (단일 항목) |
| `code` | String | 요소 코드 (예: E, E1, BUS, BUS_1) |
| `name` | String | 한글 이름 |
| `definition` | String | 정의 (null 가능) |
| `parent_code` | String | 상위 요소 코드 (upper이면 null) |

## 2. career_attributes — 직업 속성 코드 (202건)

### 도큐먼트 구조
```json
{
  "category": "ability",
  "code": "AB01",
  "name": "기술 분석",
  "definition": "새로운 방법을 고안하고 기존의 방법을 개선하기 위해서 현재 사용되는 도구와 기술을 분석한다"
```

| 필드 | 타입 | 설명 |
|------|------|------|
| `category` | String | 아래 7개 카테고리 중 하나 |
| `code` | String | 항목 코드 (예: AB01, KN05, R) |
| `name` | String | 한글 이름 |
